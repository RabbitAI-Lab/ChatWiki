# 模板列表分组功能实现计划

## 背景

用户希望在 `/templates` 页面中，将用户创建的模板和系统预置模板分开显示：
- 上方分组：**我创建的** - 用户新建的模板（可编辑、可删除）
- 下方分组：**系统模板** - 系统预置的模板（只读，不可编辑/删除）

当前模板表没有区分系统模板和用户创建模板的字段，需要添加。

## 实现方案

### 1. 数据库 Schema 修改

**文件**: `src/db/schema.ts`

在 `templates` 表添加 `isSystem` 字段：
```typescript
isSystem: integer("is_system").notNull().default(0),  // 0=用户创建, 1=系统模板
```

### 2. 种子数据更新

**文件**: `src/db/seed.ts`

给6个系统预置模板添加 `isSystem: 1` 标识。

### 3. API 更新

**文件**: `src/app/api/templates/route.ts`

- POST 创建模板时，默认 `isSystem: 0`
- GET 返回数据包含 `isSystem` 字段

**文件**: `src/app/api/templates/[id]/route.ts`

- PATCH/DELETE: 检查 `isSystem`，禁止修改/删除系统模板，返回 403 错误

### 4. 前端 - 模板列表页面

**文件**: `src/components/templates/TemplatesPageClient.tsx`

- 更新 `Template` 接口，添加 `isSystem` 字段
- 将模板数据按 `isSystem` 分组渲染
- 系统模板卡片**不显示删除按钮**
- 每个分组有独立的标题

### 5. 前端 - 模板编辑器

**文件**: `src/components/templates/TemplateEditor.tsx`

- 接收 `isSystem` 属性
- 系统模板：
  - 禁止编辑名称
  - 禁止编辑内容（编辑器设为只读）
  - 不显示删除按钮
  - 显示"只读模式"提示标签
- 用户模板：保持原有可编辑功能

### 6. 模板详情页面

**文件**: `src/app/templates/[id]/page.tsx`

- 传递 `isSystem` 字段给 `TemplateEditor`

## 关键文件

| 文件 | 修改内容 |
|-----|---------|
| `src/db/schema.ts` | 添加 `isSystem` 字段 |
| `src/db/seed.ts` | 系统模板设置 `isSystem: 1` |
| `src/app/api/templates/route.ts` | 创建时默认 `isSystem: 0`，返回包含该字段 |
| `src/app/api/templates/[id]/route.ts` | 禁止修改/删除系统模板 |
| `src/app/templates/[id]/page.tsx` | 传递 `isSystem` 字段 |
| `src/components/templates/TemplatesPageClient.tsx` | 分组展示 + 系统模板隐藏删除按钮 |
| `src/components/templates/TemplateEditor.tsx` | 系统模板只读模式 |

## 验证方式

1. 运行 `npm run db:seed` 重新初始化数据库
2. 访问 `/templates` 页面
3. 确认显示两个分组："我创建的" 和 "系统模板"
4. 系统模板卡片无删除按钮
5. 点击系统模板，进入只读模式：
   - 名称不可编辑
   - 编辑器内容不可修改
   - 无删除按钮
   - 显示"只读模式"标签
6. 创建新模板，确认出现在"我创建的"分组
7. 用户模板可正常编辑和删除
